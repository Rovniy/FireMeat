1. Использован препроцессор {LESS} для генерации кода CSS
2. Получившийся код можно использовать где угодно, лишь передав, например, в URL через хештэг имя нужной формы. Это реализовано с помощью псевдокласса CSS :target.
3. На смартфонах и планшетах эти формы работают, позиционируются и меняются. Можно еще усложнить JS код, чтобы сайт постоянно отслеживал действия пользователя - поворот дисплея, масштабирование и т.п., чтобы форма постоянно находилась по центру. Думаю, сейчас, это ни к чему.
4. Дизайнер, по моему мнению, забыл в форме регистрации продублировать введенный пароль. Считаю эту функцию просто необходимой. В форме входа еще можно было бы вставить чекбокс с функцией запоминания пользователя (через Cookies или еще как-нить.)
Так же, дизайнер забыл про интерактивность и поведение форм, полей и ссылок. Я на свой вкус сделал формы и ссылки, их отображение и поведение.
